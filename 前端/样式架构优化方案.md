# 前端样式架构优化方案

## 问题修复总结

### 原始问题
- **Sass编译错误**: `[sass] Undefined mixin` 错误
- **根本原因**: 多个Vue组件使用了SCSS混合器（如`@include text-ellipsis`），但未正确导入包含这些混合器的`variables.scss`文件
- **影响范围**: 7个组件文件存在此问题

### 修复方案

#### 1. 全局SCSS配置优化
在`vite.config.js`中添加CSS预处理器配置：

```javascript
css: {
  preprocessorOptions: {
    scss: {
      additionalData: `@import "@/styles/variables.scss";`
    }
  }
}
```

**优势**:
- 自动在所有SCSS文件中导入变量和混合器
- 避免在每个组件中手动导入
- 减少代码重复和维护成本
- 确保样式一致性

#### 2. 受影响文件清理
移除了以下文件中的手动导入语句：
- `src/views/user/MyProducts.vue`
- `src/views/user/Messages.vue`
- `src/views/user/MyOrders.vue`
- `src/views/user/OrderDetail.vue`
- `src/views/user/MyFavorites.vue`
- `src/views/admin/Users.vue`
- `src/views/auth/ForgotPassword.vue`

### 8. 主题切换实现

```javascript
// src/composables/useTheme.js
import { ref, watch } from 'vue'

const theme = ref(localStorage.getItem('theme') || 'light')

export function useTheme() {
  const setTheme = (newTheme) => {
    theme.value = newTheme
    document.documentElement.setAttribute('data-theme', newTheme)
    localStorage.setItem('theme', newTheme)
  }
  
  const toggleTheme = () => {
    setTheme(theme.value === 'light' ? 'dark' : 'light')
  }
  
  // 初始化主题
  setTheme(theme.value)
  
  // 监听系统主题变化
  const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)')
  const handleSystemThemeChange = (e) => {
    if (!localStorage.getItem('theme')) {
      setTheme(e.matches ? 'dark' : 'light')
    }
  }
  
  mediaQuery.addEventListener('change', handleSystemThemeChange)
  
  return {
    theme,
    setTheme,
    toggleTheme
  }
}
```
